Overview

This QA portfolio contains the full documentation created during my manual QA training and practice. It includes structured Test Cases, detailed Bug Reports, and notes from testing real websites (e-commerce and a government complaint system). The goal is to show how I think as a QA tester, how I design test scenarios, and how I document issues clearly for developers.

Training Summary

  • Day 1: User vs QA mindset, features analysis, test scenarios, equivalence classes, boundary values, test data design.
  • Day 2: Wrote 12 structured Test Cases for Login (web), using formal QA format (ID, preconditions, steps, expected result).
  • Day 3: Wrote 14 Bug Reports for e-commerce and government websites, with severity/priority classification.
  • Day 4: Integration into portfolio, CV update, and documentation cleanup.

Bug Reports (Web & Mobile)

Each Bug Report includes clear title, environment, steps to reproduce, expected vs actual result, and severity/priority to help developers understand impact and fix order.

BUG-001 – "Forgot Password" button does not work

Environment: Windows 10 / Chrome browser

Steps to Reproduce:

  1. Open the Login page.
  2. Click the "Forgot Password" button/link.

Expected Result: System opens the password reset page or sends the user to a reset flow.

Actual Result: Nothing happens; the button does not respond.

Severity: Medium

Priority: Medium

BUG-002 – Login button stays disabled with valid input

Environment: Windows 10 / Brave browser

Steps to Reproduce:

  1. Open the Login page.
  2. Enter a valid email.
  3. Enter a valid password.
  4. Observe the Login button state.

Expected Result: Login button becomes enabled when valid email and password are entered.

Actual Result: Login button remains disabled even with valid input.

Severity: Critical

Priority: High

BUG-003 – No error message for invalid email format

Environment: Android / Mobile

Steps to Reproduce:

  1. Open the Login page on Android mobile.
  2. Enter an invalid email format (e.g., "omar@", "omar.com").
  3. Enter a valid password.
  4. Tap the "Login" button.

Expected Result: System displays an error message indicating the email format is invalid.

Actual Result: No error message is displayed; the system does not validate the email format.

Severity: Medium

Priority: Medium

BUG-004 – Password field accepts input shorter than minimum length

Environment: Windows 10 / Chrome browser

Steps to Reproduce:

  1. Open the Login page.
  2. Enter a valid email.
  3. Enter a password shorter than the required minimum (e.g., 2–3 characters).
  4. Click the "Login" button.

Expected Result: System should show validation error about minimum password length.

Actual Result: Login request is submitted without any validation error.

Severity: Major

Priority: High

BUG-005 – "Remember Me" checkbox does not preserve login session

Environment: Windows 11 / Firefox

Steps to Reproduce:

  1. Open the Login page.
  2. Enter valid email and password.
  3. Check the "Remember Me" option.
  4. Login successfully.
  5. Close the browser and reopen the website.

Expected Result: User should remain logged in and not be asked to log in again.

Actual Result: User is logged out and must log in again.

Severity: Major

Priority: Medium

BUG-006 – Login page takes too long to load

Environment: Windows 10 / Edge browser

Steps to Reproduce:

  1. Open the browser.
  2. Navigate to the Login page.

Expected Result: Login page should load within 1–2 seconds.

Actual Result: Login page takes around 5–7 seconds to fully load.

Severity: Minor

Priority: Medium

BUG-007 – Password visibility toggle does not change visibility

Environment: Android / Chrome mobile

Steps to Reproduce:

  1. Open the Login page on Android.
  2. Enter a password.
  3. Tap the eye (visibility) icon.

Expected Result: Password should toggle between visible and hidden states.

Actual Result: Password remains hidden; icon does not function.

Severity: Minor

Priority: Medium

BUG-008 – Email field with special characters crashes or freezes page

Environment: Windows 10 / Chrome browser

Steps to Reproduce:

  1. Open the Login page.
  2. Enter only special characters in the email field (e.g., "%$#@!").
  3. Enter a valid password.
  4. Click the "Login" button.

Expected Result: System should show an "Invalid email format" error.

Actual Result: Page crashes or becomes unresponsive.

Severity: Critical

Priority: High

BUG-009 – Phone number fields accept alphabetic characters (Gov portal)

Environment: Windows 10 / Chrome browser

Page: Government Complaint System – Create New Account → Personal Information section

Steps to Reproduce:

  1. Open the "Create New Account" page.
  2. Scroll to the "Personal Information" form.
  3. Enter alphabetic characters into "Mobile Number" and "Phone Number" fields.
  4. Click "Next" or proceed in the form.

Expected Result: System should restrict input to digits only or show validation error messages.

Actual Result: System accepts alphabetic characters without any validation and allows user to proceed.

Severity: Major

Priority: High

BUG-010 – Search returns no results for existing products

Environment: Windows 10 / Chrome browser (e-commerce site)

Steps to Reproduce:

  1. Open the home page.
  2. Type a known existing product (e.g., "iPhone") in the search bar.
  3. Press Enter or click the Search button.

Expected Result: A list of relevant products matching the keyword should be shown.

Actual Result: "No results found" message appears although the product exists.

Severity: Major

Priority: High

BUG-011 – Cart total price is calculated incorrectly

Environment: Windows 10 / Chrome browser (e-commerce site)

Steps to Reproduce:

  1. Add Product A to cart (e.g., 50).
  2. Add Product B to cart (e.g., 30).
  3. Open the Cart page.

Expected Result: Cart total should show 80.

Actual Result: Cart total displays an incorrect value.

Severity: Critical

Priority: High

BUG-012 – Reviews section does not load on product page

Environment: Android / Chrome mobile (e-commerce site)

Steps to Reproduce:

  1. Open any product page that has customer reviews.
  2. Scroll down to the "Customer Reviews" section.

Expected Result: Reviews should load and be displayed.

Actual Result: Reviews section keeps loading or remains empty with no message.

Severity: Medium

Priority: Medium

BUG-013 – No error message for wrong password

Environment: Windows 11 / Edge browser

Steps to Reproduce:

  1. Open the Login page.
  2. Enter a valid email.
  3. Enter an incorrect password.
  4. Click "Login".

Expected Result: System shows an error message like "Incorrect email or password".

Actual Result: Page refreshes or stays the same with no feedback.

Severity: Major

Priority: High

BUG-014 – Price filter does not update product list

Environment: Windows 10 / Firefox (e-commerce site)

Steps to Reproduce:

  1. Open any product category page.
  2. Set a price range (e.g., 50–100) using the price filter.
  3. Apply the filter.

Expected Result: Product list should display only items within the selected price range.

Actual Result: Product list remains unchanged; items outside the range are still displayed.

Severity: Major

Priority: Medium
